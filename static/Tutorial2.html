🎓 답안 작성 가이드

좋은 답안 작성법을 배우고 다른 친구들의 생각을 살펴보세요!

🤔 첫 번째 튜토리얼 문제(변수 값 교환)에 대해 친구에게 설명한다면, 어떤 것이 가장 좋은 설명일까요?
"첫 줄은 tmp에 a를 저장하고, 둘째 줄은 a에 b를, 셋째 줄은 b에 tmp를 저장해."
"a와 b를 더하는 코드야."
"tmp를 써서 변수 값을 바꾸는 코드야."
"a와 b의 값을 서로 바꾸는(교환하는) 코드야."
❌ 너무 길어요: 코드의 동작을 그대로 설명하기보다 전체적인 '역할'을 요약하는 게 좋아요.
❌ 틀린 설명이에요: 코드는 덧셈이 아니라 값의 교환을 수행해요.
❌ 애매한 설명이에요: '값을 바꾼다'가 구체적으로 무엇을 의미하는지 명확하지 않아요.
✅ 정확하고 간결해요! 코드의 핵심 역할인 '값 교환'을 명확히 설명했어요.

👀 다른 친구들의 답변 예시

"두 변수의 값을 임시 변수를 사용해서 바꾼다."

👍 좋은 점: '임시 변수'라는 핵심 전략과 '두 변수의 값을 바꾼다'는 목적을 모두 명확하게 언급했어요.

"a의 값을 tmp에 백업하고 a에 b를 넣은 뒤, b에 백업해둔 tmp 값을 넣어 두 변수의 값을 교체함."

🤔 아쉬운 점: 장황하게 줄별 기능을 기술하여, 전체를 설명하지 못한다는 단점이 있습니다.

"a는 b가 되고 b는 a가 된다."

🤔 아쉬운 점: 두 변수가 바뀌는 것인지, 두 변수의 값이 바뀌는 것인지 애매합니다. 중의적인 표현을 지양하는 것이 바람직합니다.

📋 앞으로의 채점 기준

이 가이드는 학생들이 앞으로 코드 읽기 과제에서 답안을 작성할 때 참고할 수 있는 기준입니다.

1️⃣ 간결성
핵심 기능을 짧고 명확하게 표현했는가?
2️⃣ 정확성
코드가 실제로 하는 일과 다르게 설명하지 않았는가?
3️⃣ 명확성
누가 읽어도 오해 없이 이해할 수 있도록 구체적으로 설명했는가?
